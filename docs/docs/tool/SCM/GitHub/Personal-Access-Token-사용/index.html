    Personal Access Token 사용

    2021년 8월 13일 이후로 비밀번호 인증 대신 Personal Access Token(PAT)을 사용해야 한다. 이전에 받은 저장소를 push 해야 할 경우 PAT로 인증하지 않으면 오류가 발생하고 안내 메시지를 출력한다.

    PAT는 GitHub가 자동 생성해주는 일종의 복잡한 비밀번호이다. 사람이 직접 생성한 비밀번호는 아무래도 단순해서 브루트 포스등에 취약하게 되는데 이를 완화하기 위한 조치로 보인다.

    GitHub Docs에 관련 문서 Creating a personal access token 를 참고하여 PAT를 생성 후 비밀번호 대신 사용하면 된다.
