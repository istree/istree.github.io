    04 데이터 캐스캐이드

    Jekyll은 레이아웃을 파일마다 써주거나. _config.yml에서 scope 범위를 지정하여 일괄 적용하는 방식을 사용한다.

    11ty는 지정한 디렉토리 하위의 모든 파일에 대해 Front Matter를 병합하는 기능이 있다.

    이를 Data Cascade라고 한다.

    예를 들어 블로그의 글이 posts 디렉토리 내에 있는 경우 모든 파일에 _include/layouts/post.liquid 레이아웃을 적용한다면

    디렉토리 이름과 같은 이름으로 posts.json 파일을 만든다.

    posts/
    ├─ 2022-12-01-글1.md
    ├─ 2022-12-11-글2.md
    ├─ 2022-12-21-글3.md
    ├─ posts.json

    Front Matter를 다음과 같이 채운다.

    {
      "layout": "layouts/post.liquid"
    }

    json 외에도 여러 방법이 있지만 json 파일로 만들었다.

    디렉토리 밖에 나란히 있어도 적용 되는듯 하나 안에 넣어주는게 보기가 편하다.

    posts 내의 모든 파일에 post.liquid 레이아웃이 적용되며 하위 디렉토리까지 모두 적용된다.

    하나 이상의 파일에 공통된 내용이 있다면 최대한 Data Cascade로 묶어주는 편이 관리하기 편리하다.
