    06 스타일 및 이미지

    css, js, 이미지 파일등 11ty로 처리하지 않고 단순히 특정 경로에 복사할 필요만 있는 데이터들이 있다.

    이런 11ty의 옵션적인 설정은 config에서 처리한다.

    11ty의 config는 프로젝트 최상위 디렉토리에 .eleventy.js 파일에 작성한다.

    Jekyll로 따지면 _config.yml 이다.

    예를 들어 빌드 시 source 디렉토리 내의 assets 디렉토리와 각 디렉토리의 모든 png 파일을 복사한다면 다음과 같이 쓴다.

    module.exports = function(config) {
      // static files
      config.addPassthroughCopy("source/assets");
      config.addPassthroughCopy("source/**/*.png");
      config.setServerPassthroughCopyBehavior("copy");
    }

    Sass나 SCSS의 경우 Jekyll에서는 번들된 플러그인으로 처리한다. 11ty도 동일하게 플러그인으로 해주어야 한다.

    자세한 내용은 11ty SASS(SCSS) 사용 참고
