    npm

    Node 패키지 매니저. Node로 개발한 프로그램을 관리하는 도구이다.

    Node를 설치하면 함께 설치된다. nvm 으로 Node를 설치하면 굳이 설치할 필요가 없다.

    설치

    https://github.com/nodesource/distributions

    nodejs 바이너리 배포판과 설정 및 지원 스크립트 를 사용하기위한 설명서

    13.x 설치 예

    # 우분투 sudo 사용
    curl -sL https://deb.nodesource.com/setup_13.x | sudo -E bash -
    sudo apt-get install -y nodejs

    # 데비안 root 설치 
    curl -sL https://deb.nodesource.com/setup_13.x | bash -
    apt-get install -y nodejs

    git 저장소에 있는 모듈 설치

    npm install git+https://github.com/your/repo.git

    npmjs

    npmjs.com

    npm의 저장소이다. 기본적으로 무료이나 유료로 공개 패키지를 사용할 수 있다.

    npm에 패키지 배포 준비

    npmjs 에서는 배포시 packages.json 의 내용을 바탕으로 패키지를 게시한다. 최소한 다음 항목들의 내용은 채우는 것이 좋다.

    description

    "description": "my project"

    author

    "author": "my name"

    license

    "license": "MIT"

    repository

    "repository": {
        "type": "git",
        "url": "https://github.com/my/repo"
    }

    keywords

    "keywords": [
        "any",
        "words"
    ]

    README.md 에 해당 패키지의 사용법 및 예제 등을 적는다.

    npm에 패키지 배포

    일단 계정이 있어야 한다.

    npm 회원가입

    배포할 패키지 디렉토리에서 로그인 한다.

    npm login

    기본적으로 private이기 때문에 유료 사용자가 아니라면 오류가 발생한다. 무료 사용자라면 public 옵션을 준다.

    npm publish --access=public

    npm 패키지 갱신

    내용을 수정하여 재배포 하는 경우

    npm publish

    오류가 발생한다.

    재배포시에는 항상 버전을 올려주어야 한다.

    npm version patch

    다시 publish를 수행하면 갱신된다.
